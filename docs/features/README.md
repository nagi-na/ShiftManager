# 機能ドキュメント（features）

新しく追加する機能ごとに、仕様・設計・本番への反映手順をまとめる場所です。

## 置き方の決まり

- 1機能 = 1ファイル。`docs/features/<機能名>.md`（英小文字・ハイフン区切り推奨。例: `shift-swap.md`）。
- 各ファイルは「何を・なぜ・どう作るか」と「本番へどう反映するか（アップデート手順）」までを1枚で追えるようにする。

## 各ファイルの推奨テンプレート

```markdown
# <機能名>

## 1. 目的・背景
（何を解決する機能か。なぜ要るか）

## 2. 仕様
（画面・操作・権限・ルール。エッジケースも）

## 3. データモデルの変更
（追加/変更するモデル・フィールド。マイグレーションの有無）

## 4. 実装メモ
（変更するファイル、ビュー/URL/テンプレート/テスト）

## 5. アップデート手順（本番反映）
（git pull → migrate → collectstatic → restart など。
  破壊的変更があればバックアップ前提。2章2-4 / 13章 / 19章 を参照）

## 6. 確認項目
（動作確認・テスト・ロールバック方法）
```
