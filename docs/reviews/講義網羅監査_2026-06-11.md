# 講義資料 網羅監査レポート（2026-06-11）

- 目的: 「**講義（`docs/lecture/`）を 0章から順に適用すると、現行リポジトリと同じ機能のアプリができるか**」を、Pythonソース（再現に必須のロジック）中心に照合する。
- 方法: リポジトリの全定義（モデルのフィールド／ビュー関数／フォーム／URL／デコレータ／admin）を列挙し、講義の**増分適用後の最終形**と1つずつ突き合わせる。テンプレートは要点照合（一部は章が意図的にリポジトリ参照）。
- 結論: **コア機能のシンボルはすべて講義に存在し、致命的な再現漏れは無い。** 照合で見つかった差分のうち、機能に影響するものは本セッションで**講義を修正**し、表示文言だけの差分は「許容（記録のみ）」とした。

## 3. 許容した差分（表示文言・docstringのみ。機能は同一）

これらは**動作に影響しない**ため、記録のみで講義は変更していません（むしろ講義側の表現が分かりやすい箇所もある）。

- **`PostDeadlinePolicy` の選択肢ラベル**: 講義「全員 提出・編集不可」等 / リポジトリ「締切後は提出・編集とも不可」等。値（`lock_all` 等）と挙動は同一。
- **`ShiftRequestDay.__str__`**: 講義「{日}: 休み」 / リポジトリ「{日}: 不可」。`__str__` の表示文字のみ。
- **`ShiftPeriod.is_visible` の verbose_name**: 講義「クルーに表示」 / リポジトリ「スタッフに表示」＋help_text。ラベルのみ。
- **`edit_deadline` の help_text** ほか、各所の docstring・コメントの語句差。
- **長いテンプレートHTML**: 20・21章は方針（簡潔な追加章）により、一部の管理系テンプレート（一覧・申請・編集）を**リポジトリ参照**にしている。Pythonロジックと主要テンプレートは全文掲載済みで、残りも記述＋リポジトリ写経で再現可能。

---

## 4. 監査の限界（正直な範囲）

- **Pythonロジックの再現性**を主対象とした。モデル定義・権限・締切判定・フォーム検証・URL・ビューの分岐は**全件照合済み**。
- **テンプレートHTMLの全行一致**までは検証していない（クラス名・余白など見た目の細部は差がありうる）。機能（表示する値・リンク・フォーム項目）は要点照合で一致を確認。
- マイグレーションファイルの内容（自動生成物）は対象外。

---

## 5. 結論

修正（F-1〜F-5）適用後、**講義 0〜9章 ＋ 13〜17章 ＋ 19〜21章 を順にたどると、固定シフト・アナウンスを含む現行アプリの機能を再現できる**状態になった。残差は表示文言・docstring・一部テンプレートHTMLの細部のみで、挙動には影響しない。
