# 4章 ホーム画面

ログイン後の入口になる **ホーム（S2）** を作ります。ここはログインした人の役割によって見せ方を変えます。

- **クルー**: 自分が提出すべき期間の一覧。自分の提出状況（提出済み／未提出）と、提出・編集ボタン。
- **リーダー**: 全期間の一覧。各期間の「提出状況を見る」「確定シフト」への入口。

この章のゴール: 役割で出し分けるホームを作り、期間ごとのカードを表示する。

> 🔰 同じ `home` ビュー・同じ `home.html` の中で「リーダーかどうか」で分岐します。画面を2つに分けず、1つのテンプレート内で `{% if %}` 分けるのがDjangoらしい書き方です。

---

## 4-1. 「締切かどうか」を判定する小さな関数

ホームでも提出画面でも「この期間はもう締切か？」を何度も判定します。共通のヘルパー関数を作っておきます。

期間が「締切扱い」になるのは次のどちらか:

- リーダーが手動で締め切った（`status` が `CLOSED`）。
- 締切日時（`deadline`）を過ぎた。

ファイル: `shifts/views.py`（末尾に追記）

```python
def _is_closed(period, now) -> bool:
    """締切扱いか。手動でclosed、または締切日時を過ぎていれば締切とみなす。"""
    return period.status == ShiftPeriod.Status.CLOSED or period.deadline < now
```

> 🔰 関数名を `_`（アンダースコア）で始めているのは「このモジュール内だけで使う補助関数」という意図表示です（Pythonの慣習）。

## 4-2. ホームのビューを作り込む

3章で作った最小の `home` を、役割で出し分ける本物に置き換えます。

ファイル: `shifts/views.py`

```python
from django.contrib.auth.decorators import login_required
from django.shortcuts import render
from django.utils import timezone

from .models import ShiftPeriod, ShiftRequest


@login_required
def home(request):
    """S2 ホーム（対象期間の一覧）。役割で表示を変える。"""
    now = timezone.now()
    periods = list(ShiftPeriod.objects.all())

    # --- リーダー: 全期間を管理目線で一覧 ---
    if request.user.is_leader:
        items = [
            {"period": p, "is_closed": _is_closed(p, now)}
            for p in periods
        ]
        return render(request, "shifts/home.html", {"items": items, "is_leader": True})

    # --- クルー: 自分の提出状況つきで一覧 ---
    submitted_period_ids = set(
        ShiftRequest.objects.filter(user=request.user).values_list("period_id", flat=True)
    )
    items = []
    for p in periods:
        is_closed = _is_closed(p, now)
        submitted = p.id in submitted_period_ids
        items.append(
            {
                "period": p,
                "submitted": submitted,
                "is_closed": is_closed,
                # 締切前なら提出・編集できる
                "can_submit": not is_closed,
            }
        )
    return render(request, "shifts/home.html", {"items": items, "is_leader": False})
```

**解説**

- `ShiftPeriod.objects.all()` … 全期間を取得（ORMの基本）。`-start_date` の並び順はモデルの `Meta.ordering` で指定済みです。
- **「未提出」をどう判定するか**: 0章で決めたとおり「`ShiftRequest` の行が無い＝未提出」。そこで「自分が提出済みの期間ID集合」を先に1クエリで作り（`submitted_period_ids`）、各期間がそこに含まれるかで `submitted` を決めます。
- `values_list("period_id", flat=True)` … 提出レコードから `period_id` だけをリストで取り出すORMの書き方。`flat=True` で `[1, 3, 5]` のような単純なリストになります。
- **N+1問題を避ける**: 期間ごとに毎回DBへ「提出した？」と聞くと、期間数だけクエリが飛びます（N+1問題）。ここでは「提出済みID集合」を**1回**で取っておき、あとはメモリ上の `in` 判定で済ませています。

> 🔰 `timezone.now()` を使うのは、settings で `USE_TZ = True` にしているから。`datetime.now()` ではなくこちらを使うのがDjangoの作法です。

## 4-3. ホームのテンプレート

3章で作った仮の `home.html` を、カード一覧の本物に置き換えます。

ファイル: `shifts/templates/shifts/home.html`

```html
{% extends "base.html" %}

{% block title %}ホーム | シフト提出{% endblock %}

{% block content %}
<div class="d-flex justify-content-between align-items-center mb-3">
  <h1 class="h4 mb-0">{% if is_leader %}対象期間（管理）{% else %}シフト提出{% endif %}</h1>
  {% if is_leader %}
    <a class="btn btn-outline-secondary btn-sm" href="{% url 'manage_top' %}">管理メニュー</a>
  {% endif %}
</div>

{% if not items %}
  <div class="alert alert-secondary">
    {% if is_leader %}まだ対象期間がありません。「管理メニュー」→「期間・締切設定」から作成してください。
    {% else %}現在、提出できる期間はありません。{% endif %}
  </div>
{% endif %}

<div class="row g-3">
  {% for item in items %}
  <div class="col-12 col-md-6">
    <div class="card h-100 shadow-sm">
      <div class="card-body">
        <div class="d-flex justify-content-between align-items-start">
          <h2 class="h6 mb-1">{{ item.period }}</h2>
          {% if item.is_closed %}
            <span class="badge bg-secondary">締切</span>
          {% else %}
            <span class="badge bg-success">募集中</span>
          {% endif %}
        </div>
        <p class="text-muted small mb-2">
          期間: {{ item.period.start_date }} 〜 {{ item.period.end_date }}<br>
          締切: {{ item.period.deadline|date:"Y/m/d H:i" }}
        </p>

        {% if is_leader %}
          <a class="btn btn-primary btn-sm" href="{% url 'period_status' item.period.pk %}">提出状況を見る</a>
          <a class="btn btn-outline-secondary btn-sm" href="{% url 'confirmed_shift' item.period.pk %}">確定シフト</a>
        {% else %}
          {% if item.submitted %}
            <span class="badge bg-info text-dark mb-2">提出済み</span>
          {% else %}
            <span class="badge bg-warning text-dark mb-2">未提出</span>
          {% endif %}
          <div class="mt-2">
            {% if item.can_submit %}
              <a class="btn btn-primary btn-sm" href="{% url 'shift_submit' item.period.pk %}">
                {% if item.submitted %}編集する{% else %}提出する{% endif %}
              </a>
            {% else %}
              <a class="btn btn-outline-secondary btn-sm" href="{% url 'shift_submit' item.period.pk %}">提出済みを確認</a>
              <span class="text-muted small d-block mt-1">締切後のため編集できません</span>
            {% endif %}
            <a class="btn btn-outline-secondary btn-sm" href="{% url 'confirmed_shift' item.period.pk %}">確定シフト</a>
          </div>
        {% endif %}
      </div>
    </div>
  </div>
  {% endfor %}
</div>
{% endblock %}
```

**解説**

- `{{ item.period }}` … モデルの `__str__`（2章で定義）が呼ばれ、タイトルか「開始〜終了」が表示されます。
- `{{ item.period.deadline|date:"Y/m/d H:i" }}` … テンプレートフィルタ `date` で日時を整形。`USE_TZ` のおかげで日本時間で表示されます。
- ボタンの `href` に `{% url 'period_status' item.period.pk %}` などを書いていますが、**これらのURL（`period_status` `confirmed_shift` `shift_submit` `manage_top`）はまだ未定義**です。次の手順で「仮のURL」を用意してエラーを防ぎます。

## 4-4. リンク先を仮で用意する（スタブ）

テンプレートで参照しているURL名が未定義だと、ホームを開いた瞬間 `NoReverseMatch` になります。5章以降で本実装するまで、**仮のビューとURL**（スタブ）を置いておきます。

ファイル: `shifts/views.py`（末尾に追記）

```python
from accounts.decorators import leader_required


# --- 以下は後の章で本実装するスタブ（仮） ---

@login_required
def shift_submit(request, pk):
    return render(request, "shifts/_stub.html", {"name": "シフト提出（5章で実装）"})


@leader_required
def period_status(request, pk):
    return render(request, "shifts/_stub.html", {"name": "提出状況一覧（6章で実装）"})


@login_required
def confirmed_shift(request, pk):
    return render(request, "shifts/_stub.html", {"name": "確定シフト（7章で実装）"})


@leader_required
def manage_top(request):
    return render(request, "shifts/_stub.html", {"name": "管理メニュー（8章で実装）"})
```

ファイル: `shifts/templates/shifts/_stub.html`（新規作成）

```html
{% extends "base.html" %}
{% block content %}
<div class="alert alert-info">「{{ name }}」はこの先の章で実装します。</div>
<a class="btn btn-outline-secondary btn-sm" href="{% url 'home' %}">← ホーム</a>
{% endblock %}
```

ファイル: `shifts/urls.py`

```python
from django.urls import path

from . import views

urlpatterns = [
    path("", views.home, name="home"),
    path("periods/<int:pk>/submit/", views.shift_submit, name="shift_submit"),
    path("periods/<int:pk>/status/", views.period_status, name="period_status"),
    path("periods/<int:pk>/confirmed/", views.confirmed_shift, name="confirmed_shift"),
    path("manage/", views.manage_top, name="manage_top"),
]
```

**解説**

- `<int:pk>` … URLの一部を変数として受け取る書き方。`/periods/3/submit/` なら `pk=3` がビューに渡ります。`pk` は primary key（主キー）の慣用名です。
- スタブを用意することで「画面遷移は全部つながっているが、中身はこれから」という状態を作れます。各章でスタブを本物に差し替えていきます。

> 🔰 こうやって**先に画面の骨格と導線を通し、後で中身を肉付け**するのは実務でもよくある進め方です。常に「動く状態」を保てます。

## 4-5. 動作確認

### ✅ 役割で表示が変わることを確認

準備として、2章の管理画面（/admin）で次を用意しておくと分かりやすいです。

- リーダー1人（2章で設定済みの管理者）。
- クルー1人（/admin で「ユーザー」追加、ロール=クルー、パスワード設定、`is_active`にチェック）。
- 対象期間を1〜2件（締切日時が「未来」のものと「過去」のものがあると締切表示を確認できます）。

確認手順:

1. **リーダーでログイン** → ホームに「対象期間（管理）」と各期間の「提出状況を見る」ボタン。右上に「管理メニュー」。
2. 期間カードの「募集中／締切」バッジが、締切日時に応じて正しく出る。
3. **一度ログアウトし、クルーでログイン** → 同じ期間が「提出済み／未提出」バッジ付きで出る。未提出なら「提出する」ボタン。
4. ボタンを押すと、スタブ画面（「5章で実装します」等）に飛ぶ。

## 4-6. つまずきポイント

- **`NoReverseMatch` でホームが開けない** → テンプレートで参照しているURL名（`manage_top` など）のスタブ・URLを入れ忘れ。4-4を見直す。
- **クルーなのに管理ボタンが見える** → ビューで `is_leader` を渡し忘れ、またはテンプレートの `{% if is_leader %}` の綴り違い。
- **締切なのに「募集中」のまま** → `deadline` が未来になっている／`USE_TZ` 環境で時刻の比較がずれている。`timezone.now()` を使っているか確認。

## この章のまとめ

- ホームを役割（クルー／リーダー）で出し分けた。
- 「未提出＝行が無い」を、提出済みID集合との `in` 判定で効率よく表現した（N+1を回避）。
- `_is_closed` ヘルパーで締切判定を共通化した。
- スタブで画面遷移を先に通し、以降の章で中身を実装する形にした。
