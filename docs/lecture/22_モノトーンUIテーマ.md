# 22章 モノトーンUIテーマ（Bootstrapに1枚重ねて見た目を整える）

> 発展編（付録）。**3章（共通レイアウト `base.html`）** が終わっていれば着手できます。機能は足さず**見た目だけ**を整える章です。
> 設計のねらいは [`../features/ui-theme.md`](../features/ui-theme.md) も参照してください。

---

## 22-0. この章で作るもの

標準のBootstrap（青基調）を、**無彩色（モノトーン）で落ち着いた見た目**に変えます。ポイントは、**Bootstrap本体やテンプレートの構造は変えず、上書きCSSを1枚重ねるだけ**で全画面に効かせること。気に入らなければ `<link>` を1行外すだけで元に戻せます。

- 配色は墨（`#18181b`）×グレー×紙のような背景。
- **色で伝えていた状態**（赤=未提出、緑=募集中…）を、**濃淡・塗り/枠線・記号**で伝え直す。
- 書体は本文＝Noto Sans JP、見出し＝Noto Serif JP（明朝）。
- 仕上げに **アニメーション**（ページ遷移のスライド＋フェード・カードのせり上がり・ボタンの押し込み）を CSS だけで足す（22-5）。

> 🔰 **なぜテンプレートを触らないのか**: 状態を表す `bg-success` や `table-danger` といったクラス名はテンプレート側にたくさん書かれています。これを書き換えると修正漏れが出ます。**クラス名はそのまま**にして、CSS側で「この色はこう見せる」と読み替えれば、テーマだけを差し替えられます。

---

## 22-1. 仕組み：BootstrapはCSS変数で色を持っている

Bootstrap 5.3 は、各コンポーネントの色を **CSS変数** で持っています。たとえばボタンは `--bs-btn-bg`（背景）・`--bs-btn-hover-bg`（ホバー）など。なので**変数を上書きするだけ**で、`!important` を多用せずに見た目を変えられます。

```css
/* 例: 主ボタンを「青」から「墨」に */
.btn-primary {
  --bs-btn-bg: #161618;
  --bs-btn-border-color: #161618;
  --bs-btn-hover-bg: #2e2e33;
  --bs-btn-hover-border-color: #2e2e33;
}
```

状態色（`.bg-success` / `.bg-danger` / `.table-danger` など）のように変数を持たないものは、要素セレクタで直接上書きします（必要な箇所だけ `!important`）。

## 22-2. テーマCSSを作る（`static/css/theme.css`）

`static/css/` を作り、`theme.css` を置きます。全文はリポジトリの `static/css/theme.css` を写経してください。ここでは**考え方が分かる要点**だけ載せます。

### デザイントークン（色をまとめて定義）

```css
:root {
  --bg:        #f3f2ef;  /* 紙のような背景 */
  --surface:   #ffffff;  /* カード等の面 */
  --ink:       #18181b;  /* 墨（基本の文字・アクセント） */
  --ink-2:     #3f3f46;  /* やや淡い墨 */
  --muted:     #71717a;  /* 補足テキスト */
  --line:      #e4e3df;  /* 罫線・境界 */
  --ink-dark:  #161618;  /* ナビ・主要ボタン */
  --ink-dark-2:#2e2e33;  /* ホバー */
}
```

> 🔰 色を**名前付きの変数**にまとめておくと、後で「もう少し濃く」を1か所の変更で全体に反映できます。これが**デザイントークン**の考え方です。

### ベース（背景・文字・書体）

```css
body, body.bg-light {
  background: var(--bg) !important;
  color: var(--ink);
  font-family: "Noto Sans JP", system-ui, sans-serif;
  line-height: 1.75;
}
h1, h2, h3, h4 {
  font-family: "Noto Serif JP", serif;  /* 見出しは明朝で上品に */
  font-weight: 600;
}
```

### 状態を「色なし」で出し分ける（キモ）

色を使えないぶん、**濃い塗り＝強調 / 淡い塗り＝中立 / 白地＋枠＝注意**で区別します。

```css
/* バッジ: 募集中・提出済み・未読 などは濃い墨で目立たせる */
.badge.bg-success, .badge.bg-info, .badge.bg-danger {
  background: var(--ink) !important; color: #fff !important;
}
.badge.bg-secondary {                 /* 締切など中立は淡いグレー */
  background: #e7e6e2 !important; color: var(--ink-2) !important;
}
.badge.bg-warning {                   /* 未提出など要対応は白地に黒枠 */
  background: #fff !important; color: var(--ink) !important; border: 1px solid var(--ink);
}

/* 提出状況一覧の未提出行: 赤の代わりに濃いめグレー＋左に墨線＋太字 */
.table-danger, .table-danger > * { background: #e6e5e1 !important; color: var(--ink); }
.table-danger td:first-child { font-weight: 600; box-shadow: inset 3px 0 0 var(--ink); }
```

### フラッシュメッセージに記号を付ける

成功/失敗が**色以外でも**分かるよう、`base.html` で付ける `app-msg` クラス（22-3）に記号を足します。

```css
.app-msg::before { font-weight: 700; }
.app-msg.alert-success::before { content: "✓"; }
.app-msg.alert-danger::before  { content: "！"; }
.app-msg.alert-warning::before { content: "⚠"; }
.app-msg.alert-info::before, .app-msg.alert-secondary::before { content: "・"; }
```

このほか、ナビバー・ボタン・カード（角丸＋淡い影＋ホバーで浮く）・フォーム・アラート・フッターも同じ方針で整えます。**全文は `static/css/theme.css` を参照**してください。

## 22-3. 共通レイアウトに読み込む（`templates/base.html`）

3章で作った `base.html` の `<head>` に、**Notoフォント**と**theme.css**を足します（Bootstrapの**後**に読み込むのが大事。後勝ちで上書きするため）。

```html
{% load static %}
...
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;500;600;700&family=Noto+Serif+JP:wght@500;600;700&display=swap" rel="stylesheet">
  <link href="{% static 'css/theme.css' %}" rel="stylesheet">  {# ← Bootstrapより後に #}
```

フラッシュメッセージに `app-msg` クラスを足し（記号を付けるため）、最下部にフッターを置きます。

```html
  <main class="container py-4 flex-grow-1">
    {% if messages %}
      {% for message in messages %}
        <div class="alert app-msg alert-{{ message.tags|default:'info' }}">{{ message }}</div>
      {% endfor %}
    {% endif %}
    {% block content %}{% endblock %}
  </main>

  <footer class="app-footer">
    <div class="container py-3 text-center">SHIFT MANAGER</div>
  </footer>
```

`<body>` は縦に伸ばしてフッターを最下部に固定します。

```html
<body class="bg-light d-flex flex-column min-vh-100">
```

> ⚠️ `{% load static %}` がファイル先頭にあること、`STATIC_URL` と `static/` ディレクトリの設定（13章で行う `collectstatic` 前提）を確認してください。開発サーバー（`DEBUG=True`）では `static/` から自動配信されます。

## 22-5. アニメーションを足す（CSSだけ）

仕上げに、画面に動きを足します。**JavaScriptは使わず、`theme.css` の末尾に追記するだけ**です。やることは4つ。

1. **ページ間の遷移**を、スライド＋フェードで切り替える（旧ページが上へ抜け、新ページが下からせり上がる）。
2. ページの**中身をふわっと出現**させる。
3. **カード**（ホーム等の並ぶ列）を**下から順番にせり上げる**（stagger）。
4. **ボタン**をホバーで浮かせ、押すと沈ませる。

### ① ページ間の遷移（View Transitions API）

最近のブラウザには、**ページ移動そのものをアニメーションにする** `View Transitions API` があります。CSSに次の1行を書くだけで、リンク/ボタンでの移動が自動でクロスフェードになります（**遅延ゼロ**＝ブラウザが本来の遷移中に重ねて演出するので、操作は重くなりません）。

```css
@view-transition { navigation: auto; }
```

既定のクロスフェードを、もっと派手な「スライド＋フェード」に差し替えます。遷移中の旧ページは `::view-transition-old(root)`、新ページは `::view-transition-new(root)` という擬似要素で指せます。

```css
@keyframes vt-old { to   { opacity: 0; transform: translateY(-14px) scale(.99); } }
@keyframes vt-new { from { opacity: 0; transform: translateY(22px)  scale(.99); } }
::view-transition-old(root) { animation: vt-old .26s ease both; }
::view-transition-new(root) { animation: vt-new .40s cubic-bezier(.22,.61,.36,1) both; }
```

> 🔰 **対応していないブラウザ（Firefox等）はどうなる？** `@view-transition` を無視して**普通に遷移**するだけです。エラーにはならず、②③の出現アニメは効くので、どの環境でも“やわらかく現れる”体験になります（= プログレッシブ・エンハンスメント）。

### ② 中身の出現　③ カードのせり上がり（stagger）

`@keyframes` で「最初は透明＆少し下」を定義し、要素に `animation` を付けます。カードは**`nth-child` ごとに `animation-delay` をずらす**ことで、順番に登場させます（これが stagger）。

```css
/* ② ページの中身がふわっと出現 */
@keyframes app-fade-in { from { opacity: 0; } to { opacity: 1; } }
main.container { animation: app-fade-in .35s ease-out both; }

/* ③ カードは下から順番にせり上がる（cubic-bezierの最後を1超にすると軽く“行き過ぎ"て弾む） */
@keyframes app-rise {
  from { opacity: 0; transform: translateY(24px) scale(.97); }
  to   { opacity: 1; transform: translateY(0)    scale(1); }
}
.row.g-3 > [class*="col-"] { animation: app-rise .5s cubic-bezier(.34,1.4,.64,1) both; }
.row.g-3 > [class*="col-"]:nth-child(1) { animation-delay: .04s; }
.row.g-3 > [class*="col-"]:nth-child(2) { animation-delay: .10s; }
.row.g-3 > [class*="col-"]:nth-child(3) { animation-delay: .16s; }
.row.g-3 > [class*="col-"]:nth-child(4) { animation-delay: .22s; }
.row.g-3 > [class*="col-"]:nth-child(5) { animation-delay: .28s; }
.row.g-3 > [class*="col-"]:nth-child(6) { animation-delay: .34s; }
.row.g-3 > [class*="col-"]:nth-child(n+7) { animation-delay: .40s; }
```

> 🔰 `animation` の最後の `both` は「開始前は最初のフレーム・終了後は最後のフレームの見た目を保つ」指定。これが無いと、開始前に一瞬すべて表示されてからアニメが始まり、チラつきます。

### ④ ボタンの反応

```css
.btn { transition: transform .14s cubic-bezier(.34,1.4,.64,1), box-shadow .15s ease, background-color .15s ease; }
.btn:hover  { transform: translateY(-2px); }   /* 浮く */
.btn:active { transform: translateY(1px) scale(.94); }  /* 押すと沈む */
```

### ⑤ 「動きが苦手な人」への配慮（重要）

スマホ/PCには「**視差効果を減らす / アニメーションを最小限に**」というOS設定があります。これをONにしている人には、**アニメーションを全部止める**のが作法です。`prefers-reduced-motion` メディアクエリで一括無効化できます。

```css
@media (prefers-reduced-motion: reduce) {
  @view-transition { navigation: none; }
  *, *::before, *::after { animation: none !important; transition: none !important; }
}
```

> 全文はリポジトリの `static/css/theme.css` 末尾「アニメーション」節を参照。

### ✅ 動作確認（アニメーション）

`runserver` 後、**ハードリロード（Ctrl+F5）**してから操作します（旧CSSのキャッシュ対策）。

1. 対応ブラウザ（Chrome/Edge/Safari）で、ログイン→管理メニュー→ホームと移動すると、**ページがスライド＋フェード**で切り替わる。
2. ホームを開くと、**カードが下から順番にせり上がる**。
3. ボタンにマウスを乗せると浮き、押すと沈む。
4. OSの「視差効果を減らす」をONにすると、動きが止まる（＝配慮が効いている）。

## つまずきポイント

- **テーマが効かない**: `theme.css` を**Bootstrapより後**に読み込んでいますか。順番が逆だとBootstrapに上書きされます。`{% load static %}` の付け忘れにも注意。
- **本番だけ反映されない**: `theme.css` は**静的ファイル**です。本番（`DEBUG=False`）では `python manage.py collectstatic` が必要（13章）。Dockerでは静的が名前付きボリュームにあるため、再ビルド後に `collectstatic` を実行（19章／運用マニュアル）。ブラウザは旧CSSをキャッシュするので**ハードリロード**で確認。
- **アニメーションが動かない/カクつく**: ①ページ遷移のスライドは**View Transitions対応ブラウザ**のみ（Firefox等は通常遷移＝仕様どおり）。②出現アニメが出ないときは `@keyframes` 名と `animation` の名前が一致しているか、`both` を付けているか確認。③OSの「視差効果を減らす」がONだと意図的に止まります（22-5⑤）。
- **元に戻したい**: `base.html` の `theme.css` の `<link>` を1行コメントアウトすれば、標準Bootstrapに戻ります（テンプレートのクラス名は変えていないため）。アニメーションだけ消したいときは、`theme.css` 末尾の「アニメーション」節を削除すればOK。
