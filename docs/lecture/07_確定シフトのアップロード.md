# 7章 確定シフトのアップロード

リーダーが調整し終えた **確定シフトをPDFで公開（S5）** し、全員が閲覧できるようにします。ここで初めて「ファイルアップロード」を扱います。

この章のゴール:

- リーダーがPDFをアップロードできる（PDF以外・大きすぎるファイルは弾く）。
- 全員がアップロード済みの確定シフトを閲覧・ダウンロードできる。
- リーダーは不要なファイルを削除できる。

---

## 7-1. アップロードファイルはどこに保存されるか

1章で設定した `MEDIA_ROOT` / `MEDIA_URL` を思い出します。

```python
MEDIA_URL = 'media/'
MEDIA_ROOT = BASE_DIR / 'media'
```

- **`MEDIA_ROOT`**: アップロードされた実ファイルを置く**サーバー上のフォルダ**（`media/`）。
- **`MEDIA_URL`**: それをブラウザから参照するときの**URLの先頭**（`/media/...`）。

2章の `ConfirmedShift.file` は `FileField(upload_to=confirmed_shift_upload_path)` でした。`upload_to` の関数が保存先のサブパスを返すので、実ファイルは例えば `media/confirmed_shifts/period_3/shift.pdf` に保存されます。

開発中にこれをブラウザへ配信するため、3章で `shift_manager/urls.py` の末尾に次を入れてあります（再掲）。

```python
if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

> ⚠️ これは**開発用**の配信です。本番では Nginx 等のWebサーバーがメディアを配信します（13章）。`DEBUG=True` のときだけ有効にしているのはそのためです。

## 7-2. アップロード用フォーム

PDFだけ・サイズ上限つきにします。

ファイル: `shifts/forms.py`（追記）

先頭の import に `ConfirmedShift` を足します。

```python
from .models import TIME_CHOICES, ConfirmedShift
```

末尾にフォームを追加:

```python
MAX_PDF_SIZE = 10 * 1024 * 1024  # 10MB


class ConfirmedShiftForm(forms.ModelForm):
    class Meta:
        model = ConfirmedShift
        fields = ["file"]
        widgets = {
            "file": forms.ClearableFileInput(
                attrs={"class": "form-control", "accept": "application/pdf"}
            )
        }

    def clean_file(self):
        f = self.cleaned_data["file"]
        if not f.name.lower().endswith(".pdf"):
            raise ValidationError("PDFファイルを選択してください。")
        if f.size > MAX_PDF_SIZE:
            raise ValidationError("ファイルサイズは10MBまでにしてください。")
        return f
```

**解説**

- **`ModelForm`**: モデルから自動でフォームを作る仕組み。`fields = ["file"]` で「ファイル欄だけ」のフォームになります（`period` や `uploaded_by` はビュー側で埋めます）。
- **`clean_file()`**: 特定フィールドの個別検証は `clean_<フィールド名>` という名前のメソッドで書けます。ここでは拡張子とサイズをチェック。
- `accept="application/pdf"` … ブラウザのファイル選択でPDFを優先表示させるヒント（あくまで補助で、検証は `clean_file` が担当）。

> 🔰 拡張子チェックは「うっかり別ファイル」を防ぐ簡易策です。中身まで厳密に検証したい場合はより高度な手段が要りますが、本講座では拡張子＋サイズで十分とします。

## 7-3. 確定シフトのビュー

4章のスタブ `confirmed_shift` を本実装にします。1つのビューで「閲覧・アップロード・削除」を扱います。

ファイル: `shifts/views.py`

import に `ConfirmedShift` と `ConfirmedShiftForm` を足します。

```python
from .forms import ConfirmedShiftForm, NoteForm, ShiftDayFormSet
from .models import (
    ConfirmedShift, ShiftPeriod, ShiftRequest, ShiftRequestDay, ShiftRequestHistory,
)
```

本体:

```python
@login_required
def confirmed_shift(request, pk):
    """S5 確定シフト閲覧。全員が閲覧、リーダーはアップロード・削除ができる。"""
    period = get_object_or_404(ShiftPeriod, pk=pk)
    form = ConfirmedShiftForm()

    if request.method == "POST":
        if not request.user.is_leader:
            messages.error(request, "アップロード/削除はリーダーのみ可能です。")
            return redirect("confirmed_shift", pk=pk)

        # 削除
        delete_id = request.POST.get("delete_id")
        if delete_id:
            cs = period.confirmed_shifts.filter(pk=delete_id).first()
            if cs:
                cs.file.delete(save=False)  # 実ファイルも消す
                cs.delete()
                messages.success(request, "確定シフトを削除しました。")
            return redirect("confirmed_shift", pk=pk)

        # アップロード
        form = ConfirmedShiftForm(request.POST, request.FILES)
        if form.is_valid():
            cs = form.save(commit=False)
            cs.period = period
            cs.uploaded_by = request.user
            cs.original_name = form.cleaned_data["file"].name
            cs.save()
            messages.success(request, "確定シフトをアップロードしました。")
            return redirect("confirmed_shift", pk=pk)

    files = period.confirmed_shifts.all()
    return render(
        request,
        "shifts/confirmed.html",
        {"period": period, "files": files, "form": form},
    )
```

**解説**

- **閲覧は全員、変更はリーダーのみ**: GET（閲覧）は誰でも。POST（アップロード/削除）のときだけ `is_leader` を確認します。だから `@login_required` のままビュー内で権限を分けています。
- **`request.FILES`**: アップロードされたファイルは通常のPOSTデータとは別に `request.FILES` に入ります。フォームには `ConfirmedShiftForm(request.POST, request.FILES)` と**両方**渡すのが必須。
- **`form.save(commit=False)`**: いったんDB保存せずインスタンスだけ作り、`period` や `uploaded_by` を埋めてから `save()`。フォームに無いフィールドを補う定番です。
- **削除時の `cs.file.delete(save=False)`**: DBの行を消す前に**実ファイルも消す**。これを忘れると `media/` にゴミファイルが残ります。
- `period.confirmed_shifts` … 2章の `related_name="confirmed_shifts"` による逆参照。

> 🔰 削除を「別URL」ではなく同じ画面へのPOST（隠しフィールド `delete_id`）で実装しています。フォームのボタンで完結でき、画面遷移が単純になります。

## 7-4. 確定シフトのテンプレート

ファイル: `shifts/templates/shifts/confirmed.html`（新規作成）

```html
{% extends "base.html" %}

{% block title %}確定シフト | {{ period }}{% endblock %}

{% block content %}
<div class="d-flex justify-content-between align-items-center mb-2">
  <h1 class="h4 mb-0">確定シフト</h1>
  <a class="btn btn-outline-secondary btn-sm" href="{% url 'home' %}">← ホーム</a>
</div>
<p class="text-muted small mb-3">
  {{ period }}（{{ period.start_date }} 〜 {{ period.end_date }}）
</p>

{% if not files %}
  <div class="alert alert-secondary">まだ確定シフトは公開されていません。</div>
{% else %}
  <div class="list-group mb-3">
    {% for cs in files %}
    <div class="list-group-item d-flex justify-content-between align-items-center">
      <div>
        <a href="{{ cs.file.url }}" target="_blank">{{ cs.original_name|default:"確定シフト.pdf" }}</a>
        <div class="text-muted small">{{ cs.uploaded_at|date:"Y/m/d H:i" }} にアップロード</div>
      </div>
      {% if user.is_leader %}
      <form method="post" onsubmit="return confirm('削除しますか？');" class="m-0">
        {% csrf_token %}
        <input type="hidden" name="delete_id" value="{{ cs.pk }}">
        <button type="submit" class="btn btn-sm btn-outline-danger">削除</button>
      </form>
      {% endif %}
    </div>
    {% endfor %}
  </div>
{% endif %}

{% if user.is_leader %}
<div class="card shadow-sm">
  <div class="card-body">
    <h2 class="h6">確定シフトをアップロード（PDF）</h2>
    <form method="post" enctype="multipart/form-data">
      {% csrf_token %}
      <div class="mb-2">
        {{ form.file }}
        {% if form.file.errors %}<div class="text-danger small">{{ form.file.errors|join:" / " }}</div>{% endif %}
      </div>
      <button type="submit" class="btn btn-primary btn-sm">アップロード</button>
    </form>
  </div>
</div>
{% endif %}
{% endblock %}
```

**解説**

- **`enctype="multipart/form-data"` は必須**: ファイルを送るフォームには必ず付けます。これが無いとファイルがサーバーに届かず、`request.FILES` が空になります。
- **`{{ cs.file.url }}`**: `FileField` は `.url` でブラウザ用URL（`/media/...`）を返します。`target="_blank"` で別タブ表示。
- アップロード欄と削除ボタンは `{% if user.is_leader %}` でリーダーにだけ見せます（クルーには閲覧リンクのみ）。

> ⚠️ 「見た目で隠す」だけでは不十分。POSTを直接投げられる可能性があるので、**ビュー側でも `is_leader` を確認**しています（7-3）。表示制御とサーバー側チェックは必ず両方やります。

## 7-5. 動作確認

### ✅ アップロード→閲覧→削除

1. リーダーでログイン → 期間カードの「確定シフト」→ PDFを選んで「アップロード」。
2. 一覧にファイル名とアップロード日時が出る。リンクを開くとPDFが別タブで表示される。
3. `media/confirmed_shifts/period_<番号>/` に実ファイルが保存されていることをエクスプローラ等で確認。
4. テキストファイル等を選ぶと「PDFファイルを選択してください」と弾かれる。
5. クルーでログインし直すと、同じ画面で**閲覧はできるが**アップロード欄・削除ボタンが出ない。
6. リーダーで「削除」→ 一覧から消え、`media/` の実ファイルも消えている。

---

## 7-6. つまずきポイント

- **ファイルが届かない／`request.FILES` が空** → フォームの `enctype="multipart/form-data"` 忘れ。
- **PDFリンクが404** → `urls.py` の `static(...)` 設定漏れ、または `DEBUG=False` になっている。開発中は `DEBUG=True`。
- **アップロードできるが画像も通る** → `clean_file` の拡張子チェックを見直す。
- **削除してもファイルが残る** → `cs.file.delete(save=False)` を呼んでいない。

---

## この章のまとめ

- `FileField` ＋ `MEDIA_ROOT/URL` でファイルアップロードと配信を実装した。
- `clean_file` でPDF限定・サイズ上限を検証した。
- 閲覧は全員・変更はリーダーのみを、ビュー側チェックとテンプレート表示制御の両方で担保した。
- 削除時は実ファイルも消す、を徹底した。
