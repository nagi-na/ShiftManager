# 1章 環境構築とプロジェクト作成

設計図ができたので、開発環境を整え、Djangoプロジェクトの骨格を作ります。この章が終わると、ブラウザでDjangoの初期ページが表示できます。

---

## 1-1. 作業フォルダと仮想環境

プロジェクト用のフォルダを作り、その中に**仮想環境（venv）**を用意します。仮想環境は「このプロジェクト専用の Python とライブラリ置き場」です。OS全体を汚さずに済みます。

### フォルダ作成

```
$ mkdir shift_manager_app
$ cd shift_manager_app
```

> 以降のコマンドはすべてこの `shift_manager_app` フォルダの中で実行します。

### 仮想環境を作る・有効化する

**macOS / Linux**

```
$ python3 -m venv venv
$ source venv/bin/activate
```

**Windows（PowerShell）**

```
> py -m venv venv
> venv\Scripts\Activate.ps1
```

有効化に成功すると、プロンプトの先頭に `(venv)` と表示されます。

> ⚠️ **Windowsでスクリプト実行が拒否される場合**: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` を一度実行してから再度有効化してください。
> 🔰 仮想環境を抜けるには `deactivate`。次回作業するときは再度「有効化」コマンドを実行します。

---

## 1-2. Django のインストール

仮想環境を有効化した状態で、Django を入れます。

```
$ python -m pip install --upgrade pip
$ pip install "Django>=5.1,<5.2"
```

入ったか確認します。

```
$ python -m django --version
5.1.x
```

> 💡 **依存の記録**: 後で他のPCでも再現できるよう、`pip freeze > requirements.txt` で使用ライブラリを書き出しておくと親切です（今は任意）。

---

## 1-3. プロジェクトとアプリを作る

Django では「プロジェクト（全体設定）」の中に「アプリ（機能のまとまり）」を作ります。

### プロジェクト作成

```
$ django-admin startproject shift_manager .
```

> ⚠️ 末尾の **`.`（ドット）を忘れない**こと。これで「今のフォルダ直下に作る」意味になり、余計な入れ子フォルダができません。

### アプリ作成

設計で決めた2つの担当に分けます。

- `accounts`: ユーザー・認証・アカウント管理
- `shifts`: 期間・提出・確定シフト・管理画面

```
$ python manage.py startapp accounts
$ python manage.py startapp shifts
```

ここまでのフォルダ構成:

```
shift_manager_app/
├── manage.py
├── shift_manager/        # 設定パッケージ
│   ├── settings.py
│   ├── urls.py
│   └── ...
├── accounts/             # アプリ
├── shifts/               # アプリ
└── venv/
```

---

## 1-4. 設定（settings.py）を整える

ファイル: `shift_manager/settings.py`

### (1) アプリの登録

`INSTALLED_APPS` に作った2アプリを追記します。

```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'accounts',   # 追加
    'shifts',     # 追加
]
```

### (2) 言語とタイムゾーン

英語・UTC になっている箇所を日本向けに変えます。

```python
LANGUAGE_CODE = 'ja'        # 'en-us' から変更
TIME_ZONE = 'Asia/Tokyo'    # 'UTC' から変更
USE_I18N = True
USE_TZ = True
```

> 🔰 `USE_TZ = True` のままにします。DBには世界標準時(UTC)で保存し、表示時に日本時間へ変換されます。「保存はUTC・表示はJST」と理解しておけばOKです。

### (3) テンプレート・静的ファイル・メディアの場所

共通HTMLを置く `templates/`、CSS/JSを置く `static/`、アップロードPDFを置く `media/` を使えるようにします。

`TEMPLATES` の `DIRS` を次のように変更:

```python
TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [BASE_DIR / 'templates'],   # [] から変更
        'APP_DIRS': True,
        # ... 以下はそのまま
    },
]
```

ファイル末尾の `STATIC_URL = 'static/'` の周辺を次のように:

```python
STATIC_URL = 'static/'
STATICFILES_DIRS = [BASE_DIR / 'static']

# アップロードされたファイル（確定シフトPDF など）
MEDIA_URL = 'media/'
MEDIA_ROOT = BASE_DIR / 'media'
```

### (4) フォルダを作る

```
$ mkdir templates static media
```

> ⚠️ `STATICFILES_DIRS` に指定したフォルダが存在しないと警告が出ます。空でよいので作っておきます。

---

## 1-5. 動作確認

### ✅ 開発サーバーを起動

```
$ python manage.py runserver
```

> ⚠️ 「You have N unapplied migration(s).」という黄色い警告が出ますが、**今は無視してOK**です。データベースの作成（マイグレーション）は次章で、カスタムユーザーを定義してから行います。先にマイグレーションしてしまうと後で作り直しになるため、あえてまだ実行しません。

ブラウザで <http://127.0.0.1:8000/> を開き、ロケット（🚀）の「The install worked successfully!」ページが出れば成功です。

止めるときはターミナルで `Ctrl + C`。

---

## 1-6. つまずきポイント

- **`django-admin` が見つからない** → 仮想環境が有効（`(venv)` 表示）か確認。`pip install` し直す。
- **`python` で動かない** → macOS/Linux は `python3` の場合あり。`python manage.py ...` を `python3 manage.py ...` に。
- **ポートが使用中** → `python manage.py runserver 8001` のように別ポートで起動。

---

## この章のまとめ

- 仮想環境を作り、Django を入れた。
- プロジェクト `shift_manager` とアプリ `accounts` / `shifts` を作成。
- 設定（言語・TZ・テンプレート・static・media）を整え、初期ページを表示できた。
- マイグレーションは**まだしていない**（次章でカスタムユーザーを定義してから）。
