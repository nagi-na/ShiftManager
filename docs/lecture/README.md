# シフト提出Webアプリを作る ― Django ハンズオン講座

情報系の大学生（Pythonの基礎はある前提）が、**シフト提出Webアプリ**を設計から実装まで自分の手で作り切るための自習テキストです。各章を順に進めれば、ログイン・シフト提出・集計・管理画面まで備えた実用的なアプリが完成します。

---

## この講座で作るもの

アルバイトのシフト希望を Web で提出・集約するアプリ。

- **クルー（スタッフ）**: ログインして、週ごとのシフト希望を提出。確定シフトを閲覧。
- **リーダー**: 募集期間の作成・締切、全員の提出状況の確認、確定シフト（PDF）の公開、アカウント管理。

最終的に次の画面が動きます。

| 画面 | 内容 | 章 |
| --- | --- | --- |
| ログイン | ID/パスワード認証 | 3章 |
| ホーム | 募集中の期間一覧・自分の提出状況 | 4章 |
| シフト提出 | 各日の希望（30分単位・休み）を入力 | 5章 |
| 提出状況一覧 | 全員×各日の表・未提出を強調・変更履歴 | 6章 |
| 確定シフト | PDFのアップロード／閲覧 | 7章 |
| 管理画面 | アカウント管理・期間/締切設定 | 8章 |

さらに**発展編**で、3段階の権限・締切後ルールの選択・期間の表示/非表示・本番運用（MySQL/デプロイ/Docker）に加え、**固定シフト**（曜日別デフォルトの反映）・**アナウンス**（お知らせ投稿・未読管理）・**モノトーンUIテーマ**・**提出状況のA4 PDF出力**・**操作ログ**（admin向けの監査機能）まで広げます。

## 前提知識・環境

- **Python**: 変数・関数・クラス・リスト/辞書が読み書きできる程度（Django未経験でOK）。
- **環境**: Python 3.10 以上。エディタ（VS Code 推奨）。ターミナルの基本操作。
- **OS**: Windows / macOS / Linux いずれも可（コマンドの差異は各章で補足）。
- **データベース**: 本講座は標準の **SQLite** のみ。インストール不要で動きます。

> Web/HTML/CSS の知識は必須ではありません。必要な範囲はその都度説明します。見た目は CDN 配信の Bootstrap を使い、CSS を書く負担を減らします。

## 進め方

1. 章は**順番に**進めてください。各章は前章の続きのコードを編集します。
2. コードは**全文を掲載**しています。まずは写経して動かし、解説で「なぜそう書くか」を理解しましょう。
3. 各章の最後に **✅ 動作確認** があります。そこまで到達できたら次章へ。
4. 困ったら各章末の **つまずきポイント** を確認してください。

### 記法

- `ターミナル` のコマンドは行頭に `$` を付けます（`$` は入力しません）。
- ファイルを編集する箇所は「ファイル: `パス`」の見出しの下にコードを示します。
- 🔰 は補足、⚠️ は注意、✅ は動作確認です。
