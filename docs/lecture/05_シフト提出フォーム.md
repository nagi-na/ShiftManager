# 5章 シフト提出フォーム

このアプリの中心、**シフト希望の提出（S3）** を作ります。対象期間の各日について「出勤するなら開始〜終了時刻」を入力し、「出られない日は休み」にチェックします。

この章で扱うこと:

- **複数行をまとめて扱うフォーム（formset）** — 期間の日数分の入力をまとめて検証・保存する。
- **入力チェック（バリデーション）** — 出勤日なのに時刻が空、開始≥終了、などを弾く。
- **保存と変更履歴** — 提出するたびにスナップショットを1件残す。

> 🔰 0章で決めた仕様の確認: 時刻は **8:00〜24:00 を30分刻み**の選択。1日1枠。**出られない日だけ「休み」にチェック**（チェックが無い＝出勤）。

---

## 5-1. 時刻はなぜ「文字列」なのか（再確認）

2章で `start_time` / `end_time` を `TimeField` ではなく `CharField`（"HH:MM"）にしました。理由は **"24:00"（その日の終わり＝深夜0時）を扱いたい**から。Pythonの時刻型 `time` は 0:00〜23:59 しか表せず、24:00 を作れません。

文字列でも比較は正しく効きます。`"08:00" < "17:00"`、`"09:30" < "24:00"` のように、**常に2桁ゼロ埋め**なので辞書順がそのまま時刻順になります。バリデーションで `start >= end` を弾くのもこの性質を使います。

## 5-2. フォームを定義する

提出フォームは3つの部品でできています。

- `ShiftDayForm` … 1日分（休み・開始・終了）。
- `ShiftDayFormSet` … それを日数分まとめたもの（**formset**）。
- `NoteForm` … 備考欄。

ファイル: `shifts/forms.py`（新規作成）

```python
from django import forms
from django.core.exceptions import ValidationError

from .models import TIME_CHOICES

# 空（未選択）を先頭に加えた時刻の選択肢
TIME_FIELD_CHOICES = [("", "----")] + TIME_CHOICES


class ShiftDayForm(forms.Form):
    """各日の希望（1日1枠）。"""

    work_date = forms.DateField(widget=forms.HiddenInput)
    is_day_off = forms.BooleanField(
        required=False,
        label="休み",
        widget=forms.CheckboxInput(attrs={"class": "form-check-input dayoff-toggle"}),
    )
    start_time = forms.ChoiceField(
        required=False,
        label="開始",
        choices=TIME_FIELD_CHOICES,
        widget=forms.Select(attrs={"class": "form-select form-select-sm time-input"}),
    )
    end_time = forms.ChoiceField(
        required=False,
        label="終了",
        choices=TIME_FIELD_CHOICES,
        widget=forms.Select(attrs={"class": "form-select form-select-sm time-input"}),
    )

    def clean(self):
        cleaned = super().clean()
        # 「休み」にチェックが無い日＝出勤する日なので、時刻が必須。
        if not cleaned.get("is_day_off"):
            start = cleaned.get("start_time")
            end = cleaned.get("end_time")
            if not start or not end:
                raise ValidationError(
                    "出勤する日は開始・終了時刻を選択してください（休みの日は「休み」にチェック）。"
                )
            if start >= end:
                raise ValidationError("開始時刻は終了時刻より前にしてください。")
        return cleaned


class BaseShiftDayFormSet(forms.BaseFormSet):
    """各日フォームの「集合」検証。隠しフィールド(work_date)の改ざんに備える。

    各フォームの clean は1行ずつしか見られないので、行をまたいだチェック
    （日付の重複・期間外）はフォームセット側の clean で行う。
    """

    def __init__(self, *args, valid_dates=None, **kwargs):
        # 期間内の日付の集合。None のときは範囲チェックをしない。
        self.valid_dates = set(valid_dates) if valid_dates is not None else None
        super().__init__(*args, **kwargs)

    def clean(self):
        super().clean()
        if any(self.errors):
            return
        seen = set()
        for form in self.forms:
            work_date = form.cleaned_data.get("work_date")
            if work_date is None:
                continue
            if work_date in seen:
                raise ValidationError(
                    "同じ日付が重複しています。画面を開き直してから提出してください。"
                )
            seen.add(work_date)
            if self.valid_dates is not None and work_date not in self.valid_dates:
                raise ValidationError(
                    "対象期間外の日付が含まれています。画面を開き直してから提出してください。"
                )


ShiftDayFormSet = forms.formset_factory(
    ShiftDayForm, formset=BaseShiftDayFormSet, extra=0
)


class NoteForm(forms.Form):
    note = forms.CharField(
        required=False,
        label="備考・連絡事項",
        widget=forms.Textarea(attrs={"rows": 3, "class": "form-control"}),
    )
```

**解説**

- **`work_date` は `HiddenInput`**: その行がどの日付かをフォームに埋め込みます。画面には出さず、送信時に一緒に返ってきます。
- **`is_day_off` は `required=False`**: チェックボックスは「チェックしない＝出勤」が普通の状態。required にすると常にチェックを強制してしまうので外します。
- **`start_time` / `end_time` は `ChoiceField`**: 2章の `TIME_CHOICES` を選択肢に使い、先頭に空欄 `("", "----")` を足しています。これで「未選択」を表現できます。
- **`clean()` でのチェックがキモ**: 「休みでない（＝出勤する）日」だけ、開始・終了が両方そろっているか、開始 < 終了かを検証します。休みの日は時刻不要なのでチェックしません。
- **`formset_factory(ShiftDayForm, formset=BaseShiftDayFormSet, extra=0)`**: `ShiftDayForm` を**何行でも**扱えるようにする仕組み。`extra=0` は「空の追加行を作らない」（行数はこちらで日数分用意するため）。`formset=BaseShiftDayFormSet` で、行をまたいだ検証を足しています。
- **`BaseShiftDayFormSet.clean()` はなぜ必要か**: `work_date` は画面に出さない `HiddenInput` ですが、送信されるのはただのテキストなので、開発者ツール等で**値を書き換えて送れます**（普通に操作する限り起きません）。同じ日付を2回送られると保存時にDBのユニーク制約に当たって500、期間外の日付を送られると不正データになります。これを「重複なし・期間内」と検証して**フォームエラーで弾く**ための集合検証です。`valid_dates` を渡したときだけ期間内チェックをします。

> 🔰 **formset とは**: 同じフォームを縦に並べて一括処理する仕組み。各行に `day-0-...` `day-1-...` のように連番の名前が付き、`management_form`（行数などの管理情報）と一緒に送信されます。テンプレートで `{{ formset.management_form }}` を必ず出すのはこのためです。

## 5-3. 提出ビューの全体像

提出画面は「同じURLが GET（表示）と POST（保存）の両方を受ける」よくある形です。流れはこうです。

1. 期間を取得。締切後なら**閲覧のみ（read_only）**にする。
2. **POST かつ編集可能** なら → フォームを検証し、OKなら保存してホームへ。
3. **それ以外（GET）** なら → 既存の提出があればその内容を、なければ空を初期表示。

ファイル: `shifts/views.py`（importとヘルパーを追加）

まず先頭の import を次のように増やします。

```python
from datetime import timedelta

from django.contrib import messages
from django.contrib.auth.decorators import login_required
from django.db import transaction
from django.shortcuts import get_object_or_404, redirect, render
from django.utils import timezone

from accounts.decorators import leader_required

from .forms import NoteForm, ShiftDayFormSet
from .models import ShiftPeriod, ShiftRequest, ShiftRequestDay, ShiftRequestHistory

WEEKDAYS_JP = ["月", "火", "水", "木", "金", "土", "日"]
```

そして、日付範囲を作る小さなヘルパーを末尾に足します。

```python
def _date_range(start, end):
    """start〜end（両端含む）の日付リスト。"""
    return [start + timedelta(days=i) for i in range((end - start).days + 1)]
```

## 5-4. 提出ビュー本体

4章で置いた `shift_submit` のスタブを、本実装に差し替えます。

ファイル: `shifts/views.py`

```python
@login_required
def shift_submit(request, pk):
    """S3 シフト提出画面。各日1枠の希望を入力・保存する。"""
    period = get_object_or_404(ShiftPeriod, pk=pk)
    now = timezone.now()

    is_closed = _is_closed(period, now)
    read_only = is_closed  # 締切後は閲覧のみ

    req = ShiftRequest.objects.filter(period=period, user=request.user).first()
    submitted = req is not None
    dates = _date_range(period.start_date, period.end_date)

    if request.method == "POST" and not read_only:
        # valid_dates=dates で「期間内の日付か」も検証する（隠しフィールド改ざん対策）
        formset = ShiftDayFormSet(request.POST, prefix="day", valid_dates=dates)
        note_form = NoteForm(request.POST)
        if formset.is_valid() and note_form.is_valid():
            _save_submission(request.user, period, formset, note_form)
            messages.success(request, "シフト希望を保存しました。")
            return redirect("home")
    else:
        # 既存の提出があれば日付ごとに初期値へ展開
        existing = {d.work_date: d for d in req.days.all()} if req else {}
        initial = [
            {
                "work_date": dt,
                "is_day_off": (not existing[dt].is_available) if dt in existing else False,
                "start_time": existing[dt].start_time if dt in existing else None,
                "end_time": existing[dt].end_time if dt in existing else None,
            }
            for dt in dates
        ]
        formset = ShiftDayFormSet(initial=initial, prefix="day")
        note_form = NoteForm(initial={"note": req.note if req else ""})

    # 締切後は入力欄を無効化（見た目も操作不可に）
    if read_only:
        for form in formset.forms:
            for field in form.fields.values():
                field.widget.attrs["disabled"] = True
        note_form.fields["note"].widget.attrs["disabled"] = True

    rows = [
        {"form": form, "date": dt, "weekday": WEEKDAYS_JP[dt.weekday()]}
        for form, dt in zip(formset.forms, dates)
    ]
    context = {
        "period": period,
        "formset": formset,
        "note_form": note_form,
        "rows": rows,
        "read_only": read_only,
        "is_closed": is_closed,
        "submitted": submitted,
        "request_obj": req,
    }
    return render(request, "shifts/submit.html", context)
```

**解説**

- `get_object_or_404(ShiftPeriod, pk=pk)` … 該当期間が無ければ自動で404を返す定番。存在チェックを自分で書かずに済みます。
- `prefix="day"` … この formset のフィールド名を `day-0-...` のように前置き。1ページに複数フォームを置くときの衝突防止です。GETとPOSTで同じ prefix を使うのが重要。
- **初期値の展開**: 既存提出の各日を `{日付: 日データ}` の辞書にしておき、日付ごとに「休みか」「開始/終了」を初期値に詰めます。`is_available` の反転に注意（DBは「出勤可」、画面は「休み」なので `not`）。
- **read_only の無効化**: 締切後はすべてのフィールドに `disabled` を付けて触れなくします。
- `rows` … テンプレートで「日付・曜日・その行のフォーム」をまとめて回せるように整形。`zip` で formset の各フォームと日付を対応づけます。

## 5-5. 保存処理（履歴つき）

提出を保存するヘルパーを作ります。ポイントは2つ:

1. **作り直し方式**: 各日データは「全消し → 入れ直し」にします（差分更新より単純で確実）。
2. **履歴を残す**: 保存のたびに、その時点の内容をJSONで `ShiftRequestHistory` に1件積みます。

ファイル: `shifts/views.py`

```python
def _save_submission(user, period, formset, note_form):
    """提出内容を保存し、変更履歴スナップショットを1件追加する。"""
    with transaction.atomic():
        req, _ = ShiftRequest.objects.get_or_create(period=period, user=user)
        req.note = note_form.cleaned_data["note"]
        req.save()

        req.days.all().delete()
        day_objs = []
        for form in formset:
            cd = form.cleaned_data
            avail = not cd.get("is_day_off", False)
            day_objs.append(
                ShiftRequestDay(
                    request=req,
                    work_date=cd["work_date"],
                    is_available=avail,
                    start_time=cd.get("start_time") if avail else None,
                    end_time=cd.get("end_time") if avail else None,
                )
            )
        ShiftRequestDay.objects.bulk_create(day_objs)

        last = req.histories.order_by("-version_no").first()
        version_no = (last.version_no + 1) if last else 1
        snapshot = {
            "note": req.note,
            "days": [
                {
                    "work_date": d.work_date.isoformat(),
                    "is_available": d.is_available,
                    "start_time": d.start_time or None,
                    "end_time": d.end_time or None,
                }
                for d in day_objs
            ],
        }
        ShiftRequestHistory.objects.create(
            request=req, version_no=version_no, snapshot=snapshot
        )
```

**解説**

- **`transaction.atomic()`**: この中の処理を「全部成功 or 全部取り消し」にします。途中でエラーが出ても中途半端なデータが残りません。
- **`get_or_create`**: 提出ヘッダ（`ShiftRequest`）が無ければ作り、あれば取得。「新規提出」と「編集」を1行で扱えます。
- **休みの扱い**: `is_day_off` が True の日は `is_available=False`、時刻は `None` で保存します。
- **`bulk_create`**: 日データをまとめて1回のSQLで挿入（高速）。
- **version_no**: 既存履歴の最大版＋1。初回は1。`ShiftRequestHistory` の `(request, version_no)` 一意制約と整合します。
- **snapshot は素のPython型だけ**: 日付は `isoformat()` で文字列化してJSONに収めます（`JSONField` は日付型をそのままは入れられないため）。

> 🔰 「なぜ全消し→入れ直し？」… 各日を1件ずつ「変更/追加/削除」と判定するより、まとめて入れ替えるほうがバグりにくいからです。件数も7件程度と少なく、性能上も問題ありません。

## 5-6. 提出画面のテンプレート

ファイル: `shifts/templates/shifts/submit.html`（新規作成）

```html
{% extends "base.html" %}

{% block title %}シフト提出 | {{ period }}{% endblock %}

{% block content %}
<div class="d-flex justify-content-between align-items-center mb-2">
  <h1 class="h4 mb-0">シフト希望の提出</h1>
  <a class="btn btn-outline-secondary btn-sm" href="{% url 'home' %}">← ホーム</a>
</div>
<p class="text-muted small mb-3">

  締切: {{ period.deadline|date:"Y/m/d H:i" }}
</p>

{% if read_only %}
  <div class="alert alert-secondary">締切済みのため、内容は閲覧のみです（編集できません）。</div>
{% endif %}

<form method="post" novalidate>
  {% csrf_token %}
  {{ formset.management_form }}

  {% if formset.non_form_errors %}
    <div class="alert alert-danger">{{ formset.non_form_errors }}</div>
  {% endif %}

  <p class="text-muted small mb-2">

  </p>

  <div class="card shadow-sm mb-3">

            <th>終了</th>
          </tr>
        </thead>
        <tbody>
          {% for row in rows %}
          {% with form=row.form %}
          <tr class="{% if form.non_field_errors %}table-danger{% endif %}">
            <td class="text-nowrap">
              {{ form.work_date }}
              {{ row.date|date:"n/j" }}（{{ row.weekday }}）
            </td>
            <td class="text-center">{{ form.is_day_off }}</td>
            <td>{{ form.start_time }}</td>
            <td>{{ form.end_time }}</td>
          </tr>
          {% if form.non_field_errors %}
          <tr class="table-danger">
            <td colspan="4" class="text-danger small">{{ form.non_field_errors|join:" / " }}</td>
          </tr>
          {% endif %}
          {% endwith %}
          {% endfor %}
        </tbody>
      </table>
    </div>
  </div>

  <div class="mb-3">
    <label class="form-label">{{ note_form.note.label }}</label>
    {{ note_form.note }}
    <div class="form-text">出られない日の理由などを記入できます。</div>
  </div>

  {% if not read_only %}
    <button type="submit" class="btn btn-primary">この内容で提出する</button>
  {% endif %}
  {% if submitted and request_obj %}
    <a class="btn btn-outline-secondary ms-2" href="{% url 'request_history' request_obj.pk %}">変更履歴を見る</a>
  {% endif %}
</form>

<script>
  // 「休み」チェックに応じて時刻入力の有効/無効を切り替える
  // チェックあり(=休み) → 時刻を無効化・クリア。チェックなし(=出勤) → 時刻を有効化。
  function syncRow(checkbox) {
    var tr = checkbox.closest('tr');
    var times = tr.querySelectorAll('.time-input');
    times.forEach(function (t) {
      t.disabled = checkbox.checked;
      if (checkbox.checked) { t.value = ''; }
    });
  }
  document.querySelectorAll('.dayoff-toggle').forEach(function (cb) {
    if (!cb.disabled) {
      syncRow(cb);
      cb.addEventListener('change', function () { syncRow(cb); });
    }
  });
</script>
{% endblock %}
```

**解説**

- **`{{ formset.management_form }}` は必須**: これが無いと「ManagementForm data is missing」エラーで保存できません。formset の行数などをサーバーに伝える隠しフィールドです。
- `{% with form=row.form %}` … 各行のフォームを `form` という短い名前で扱うための書き方。
- **エラー表示**: 各行の `non_field_errors`（`clean()` で出したエラー）を、その行の下に赤く出します。
- **末尾の `<script>`**: 「休み」チェックを入れたらその行の時刻を無効化＆クリア、外したら有効化、という小さなUI制御。サーバー側の `clean()` でも同じルールを検証しているので、JSはあくまで使い勝手向上です（JSを切っても検証は効く）。
- `request_obj` への変更履歴リンク（`request_history`）は次章で実装します。

> 🔰 `novalidate` … ブラウザ標準の入力チェックを止め、検証をサーバー側（Djangoの `clean()`）に一本化しています。エラー文言を日本語で統一できます。

## 5-7. 動作確認

### ✅ 提出してみる

1. クルーでログイン → ホームの未提出期間で「提出する」。
2. ある日を「休み」にチェック → その行の時刻が**自動で無効**になることを確認。
3. 出勤日の時刻を空のまま提出 → 「開始・終了時刻を選択してください」と赤エラー。
4. 開始＞終了で提出 → 「開始時刻は終了時刻より前に」と赤エラー。
5. 正しく入れて提出 → ホームに戻り、その期間が「提出済み」になる。
6. もう一度開く → さっきの内容が初期表示される（編集できる）。
7. /admin の「シフト希望」を見ると、各日データと履歴(version 1, 2...)が増えているのが分かる。

## 5-8. つまずきポイント

- **「ManagementForm data is missing」** → テンプレートに `{{ formset.management_form }}` が無い。
- **保存しても各日が空** → GETとPOSTで `prefix="day"` が食い違っている。両方そろえる。
- **`disabled` の項目が保存で消える** → `disabled` なフィールドはブラウザが送信しません。本アプリは「締切後＝read_onlyでそもそも保存しない」のでOK。出勤日の時刻をJSで一時的にdisabledにしている点は、提出時にチェックを外す運用で問題になりません（休みの日は時刻不要なため）。
- **JSONFieldにdatetimeを入れてエラー** → `isoformat()` で文字列化してから入れる。
