# 3章 認証とログイン

ここからは自分の画面を作っていきます。最初は **ログイン／ログアウト** です。シフト提出アプリは「誰が見ているか」が常に重要なので、認証を土台として先に固めます。

この章のゴール:

- ログインしていない人は自動でログイン画面に飛ばす。
- ログイン／ログアウトができる。
- 全画面で共通の見た目（ヘッダー付きレイアウト）を使えるようにする。
- 「リーダーだけが入れる画面」を作るための仕組み（デコレータ）を用意する。

> 🔰 Djangoには**認証機能が標準で入っています**（`django.contrib.auth`）。パスワードのハッシュ化、ログイン状態の保持（セッション）、ログイン必須化などを自前で書く必要はありません。私たちは「画面（テンプレート）」と「どのURLに割り当てるか」を用意するだけです。

---

## 3-1. 認証の流れを理解する

Djangoのログインはこう動きます。

1. ユーザーがログイン画面でID/パスワードを送信。
2. Djangoが正しいか照合し、正しければ**セッション**（ログイン状態）を発行。以後ブラウザはCookieでそれを持ち歩く。
3. 各ビューで `request.user` を見れば「今アクセスしている人」が分かる。未ログインなら `request.user.is_authenticated` が `False`。

「ログイン必須」にしたいビューには `@login_required` を付けるだけで、未ログイン時はログイン画面へリダイレクトされます。

## 3-2. 設定（settings.py）に認証の行き先を書く

ファイル: `shift_manager/settings.py`（末尾あたりに追記）

```python
# 認証まわりのリダイレクト先（URLの name で指定）
LOGIN_URL = 'login'            # 未ログインで保護画面に来たらここへ
LOGIN_REDIRECT_URL = 'home'    # ログイン成功後の行き先
LOGOUT_REDIRECT_URL = 'login'  # ログアウト後の行き先
```

- `'login'` `'home'` は、これから定義するURLの**名前（name）**です。文字列で名前を指定しておくと、URLのパスが変わっても設定を直さずに済みます。

> ⚠️ ここで指定した名前のURLが**存在しないと**、ログイン/ログアウト時に `NoReverseMatch` エラーが出ます。次の手順でちゃんと `login` / `home` を定義するのでセットで進めてください。

## 3-3. URLを繋ぐ

### プロジェクトのURL

ファイル: `shift_manager/urls.py`

```python
"""URL configuration for shift_manager project."""
from django.conf import settings
from django.conf.urls.static import static
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('accounts.urls')),
    path('', include('shifts.urls')),
]

# 開発時のみ: アップロードされたメディア(確定シフトPDF等)を配信
if settings.DEBUG:
    urlpatterns += static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
```

- `include('accounts.urls')` で、各アプリのURL定義を取り込みます。アプリごとにURLを分けると見通しが良くなります。
- 末尾の `static(...)` は7章でアップロードPDFを表示するときに使います（今は気にしなくてOK）。

### accounts アプリのURL（ログイン／ログアウト）

ファイル: `accounts/urls.py`（新規作成）

```python
from django.contrib.auth import views as auth_views
from django.urls import path

from . import views

urlpatterns = [
    path(
        "login/",
        auth_views.LoginView.as_view(
            template_name="registration/login.html",
            redirect_authenticated_user=True,
        ),
        name="login",
    ),
    path("logout/", auth_views.LogoutView.as_view(), name="logout"),
]
```

- `LoginView` / `LogoutView` は **Django標準のビュー**。ログイン処理そのものは書かず、使うテンプレートだけ指定します。
- `redirect_authenticated_user=True` … すでにログイン済みの人がログイン画面に来たら、ホームへ飛ばします。

> 🔰 `accounts/views.py` はまだ空（`startapp` 直後の状態）でOKです。アカウント管理のビューは8章で書きます。

## 3-4. ログイン画面のテンプレート

共通レイアウト（`base.html`）をまだ作っていないので、先に最小の `base.html` を作り、その後ログイン画面を作ります。

### 共通レイアウト base.html

ファイル: `templates/base.html`（新規作成）

```html
{% load static %}
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>{% block title %}シフト提出{% endblock %}</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body class="bg-light">
  {% if user.is_authenticated %}
  <nav class="navbar navbar-expand-lg navbar-dark bg-primary">
    <div class="container">
      <a class="navbar-brand" href="{% url 'home' %}">シフト提出</a>
      <div class="d-flex align-items-center gap-3">
        <span class="navbar-text text-white">
          {{ user.name|default:user.username }}
          {% if user.is_leader %}<span class="badge bg-light text-dark">リーダー</span>{% endif %}
        </span>
        <form method="post" action="{% url 'logout' %}" class="m-0">
          {% csrf_token %}
          <button type="submit" class="btn btn-sm btn-outline-light">ログアウト</button>
        </form>
      </div>
    </div>
  </nav>
  {% endif %}

  <main class="container py-4">
    {% if messages %}
      {% for message in messages %}
        <div class="alert alert-{{ message.tags|default:'info' }}">{{ message }}</div>
      {% endfor %}
    {% endif %}
    {% block content %}{% endblock %}
  </main>
</body>
</html>
```

**解説**

- **テンプレート継承**: `base.html` が「枠」、各画面が `{% block content %}` の中身を埋める形にします。これで全画面が同じヘッダー・同じ見た目になります。
- `{% if user.is_authenticated %}` … ログイン時だけナビバーを出します（ログイン画面ではナビバーを出さない）。
- `user.name` / `user.is_leader` … 2章で作ったカスタムUserのフィールド・プロパティをそのまま使えます。
- **messages**: ビューから出す「保存しました」等の通知をここでまとめて表示します。Bootstrapの `alert-success` / `alert-error` などに対応します。
- `{% csrf_token %}` … フォーム送信時の**なりすまし対策**。Djangoでは `<form method="post">` には必ず付けます（無いと送信が403で弾かれます）。ログアウトを**POSTのフォーム**にしているのは、Django5で `GET` でのログアウトが廃止されたためです。

> 🔰 ログアウトをリンク（`<a>`）ではなくフォームのボタンにしている点に注意。Django 5 では `LogoutView` は POST のみ受け付けます。

### ログイン画面

ファイル: `templates/registration/login.html`（新規作成）

```html
{% extends "base.html" %}

{% block title %}ログイン | シフト提出{% endblock %}

{% block content %}
<div class="row justify-content-center">
  <div class="col-12 col-sm-8 col-md-5">
    <div class="card shadow-sm">
      <div class="card-body p-4">
        <h1 class="h4 mb-4 text-center">シフト提出 ログイン</h1>

        {% if form.errors %}
          <div class="alert alert-danger">ログインIDまたはパスワードが正しくありません。</div>
        {% endif %}

        <form method="post">
          {% csrf_token %}
          <div class="mb-3">
            <label for="{{ form.username.id_for_label }}" class="form-label">ログインID</label>
            <input type="text" name="{{ form.username.html_name }}"
                   id="{{ form.username.id_for_label }}"
                   class="form-control" autofocus required>
          </div>
          <div class="mb-3">
            <label for="{{ form.password.id_for_label }}" class="form-label">パスワード</label>
            <input type="password" name="{{ form.password.html_name }}"
                   id="{{ form.password.id_for_label }}"
                   class="form-control" required>
          </div>
          <input type="hidden" name="next" value="{{ next }}">
          <button type="submit" class="btn btn-primary w-100">ログイン</button>
        </form>

        <p class="text-muted small mt-3 mb-0 text-center">
          パスワードが分からない場合はリーダーに連絡してください。
        </p>
      </div>
    </div>
  </div>
</div>
{% endblock %}
```

**解説**

- `LoginView` はフォームを `form` という名前でテンプレートに渡します。`form.username` / `form.password` がそのフィールドです。
- `{{ form.username.html_name }}` … `name` 属性（Djangoが期待する送信名）を埋め込んでいます。手書きの `<input>` でもこれを使えばDjango側で正しく受け取れます。
- `next` … 「ログインしようとして弾かれた元のURL」。ログイン後にそこへ戻すための隠しフィールドです。

> 🔰 テンプレートの置き場所がポイント。ログイン画面は慣習で `templates/registration/login.html` に置きます（`LoginView` のデフォルト名でもあります）。

## 3-5. 最小のホームと、リーダー判定デコレータ

ログイン後の行き先 `home` を用意します。中身は4章で作り込むので、まずは表示されるだけの最小版です。

### shifts のビュー（最小ホーム）

ファイル: `shifts/views.py`

```python
from django.contrib.auth.decorators import login_required
from django.shortcuts import render


@login_required
def home(request):
    """S2 ホーム。中身は4章で作り込む。"""
    return render(request, "shifts/home.html", {})
```

### shifts のURL

ファイル: `shifts/urls.py`（新規作成）

```python
from django.urls import path

from . import views

urlpatterns = [
    path("", views.home, name="home"),
]
```

### 仮のホームテンプレート

ファイル: `shifts/templates/shifts/home.html`（新規作成）

```html
{% extends "base.html" %}

{% block title %}ホーム | シフト提出{% endblock %}

{% block content %}
<h1 class="h4">ログインできました</h1>
<p class="text-muted">この画面は4章で作り込みます。</p>
```

### リーダー専用画面のためのデコレータ

「リーダーだけが入れる画面」を後の章で何度も作ります。毎回チェックを書くのは面倒なので、**デコレータ**にまとめます。

ファイル: `accounts/decorators.py`（新規作成）

```python
from functools import wraps

from django.contrib import messages
from django.shortcuts import redirect


def leader_required(view):
    """ログイン済み かつ リーダーのみ許可。それ以外はホームへ戻す。"""

    @wraps(view)
    def wrapper(request, *args, **kwargs):
        if not request.user.is_authenticated:
            return redirect("login")
        if not request.user.is_leader:
            messages.error(request, "この画面はリーダーのみ利用できます。")
            return redirect("home")
        return view(request, *args, **kwargs)

    return wrapper
```

**解説**

- デコレータは「ビュー関数を包んで、前後に処理を足す」仕組み。ここでは本来の処理の**前に**「ログイン済みか？」「リーダーか？」を確認しています。
- 未ログインなら `login` へ、ログイン済みだがリーダーでなければメッセージを出して `home` へ戻します。
- `@wraps(view)` … 包んでも元の関数名などを保つためのお約束です。

> 🔰 `@login_required` は「ログインさえしていれば誰でも」。`@leader_required` は「ログイン済み**かつ**リーダー」。用途で使い分けます。

## 3-6. 動作確認

### ✅ ログイン／ログアウトを試す

```
$ python manage.py runserver
```

1. <http://127.0.0.1:8000/> にアクセス → 未ログインなので **ログイン画面にリダイレクト**される。
2. 2章で作った管理者（リーダーに設定済み）のID/パスワードでログイン → **ホーム**（「ログインできました」）が表示される。
3. ヘッダーに自分の名前と「リーダー」バッジが出る。
4. 「ログアウト」ボタン → ログイン画面に戻る。
5. わざと間違ったパスワードでログイン → 赤いエラーメッセージが出る。

ここまで動けば認証の土台は完成です。

## 3-7. つまずきポイント

- **`NoReverseMatch: 'home'`／`'login'`** → settings の `LOGIN_REDIRECT_URL` 等で指定した名前のURLが未定義。`urls.py` の `name=` と一致しているか確認。
- **ログアウトで 405 / エラー** → Django 5 はログアウトが POST 限定。`<a href>` ではなく、`base.html` のようにフォーム＋ボタンにする。
- **POSTで 403 (CSRF)** → フォーム内に `{% csrf_token %}` を入れ忘れ。
- **ログインしても弾かれる** → そのユーザーが `is_active=False` だと拒否されます。/admin で有効か確認。
